Background
Cyera empowers security and data governance teams to detect, investigate, and remediate data risks at scale. The Automation Engine reduces manual effort and accelerates response times through policy-triggered workflows.
What We're Building
A native interface for managing remediation automation rules — enabling users to define automated actions when specific issues are detected.
·  Trigger — what event starts the rule
·  Conditions — which events qualify
·  Actions — what happens next
Two Entry Points
Via a Specific Policy
Contextual — the user is already investigating a policy and wants to automate its response.
Via the Automation Center
Deliberate — the user proactively builds and manages the organisation's automation strategy.

Discovery & Research
Key Findings
01
Analysts see the same violations repeatedly and remediate them manually each time.
The trigger for automation is always present — but the entry point to create a rule is disconnected from where the work happens.
02
Users won't activate automation they don't understand or trust.
Before enabling a rule, analysts need to see what it will do, why it applies, and what the impact will be — confidence must come before commitment.

Design Rationale
Why natural language?
Node-Based Interfaces
Zapier, n8n, Make
— Require users to learn a visual mental model before they can build anything
— Abstract away the problem — analysts must translate risk thinking into nodes and connectors
— High setup investment, designed for technical builders not security analysts
Classic Input Fields
Step-by-step form wizards
— Force users to think in system terms (trigger, condition, action) before they know what they want
— Discovery is impossible — you need to know the exact field and value to configure
— Multiple steps create friction at exactly the moment adoption could happen
Natural Language
Chosen
AI-assisted, intent-first
✓  Analysts describe the problem the way they already think about it
✓  The system translates intent into a rule — and shows its reasoning
✓  Surfaces automation at the moment of discovery, not after
✓  Confidence through transparency — the rule is readable before activation
